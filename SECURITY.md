# Security Policy

## Reporting a Vulnerability

We advocate for **Responsible Disclosure**. If you discover a vulnerability, please report it privately.

### Reporting Process

- **GitHub Security Advisories (Preferred)**: Navigate to the **[Security tab](https://github.com/wtransport/wtransport-org/security)** and click **"Report a vulnerability"** to open an encrypted draft.

- **Email (Alternative)**: Send a message to `security@wtransport.org` with the subject `[SECURITY] WTransport Org Vulnerability Report`.

### Report Contents

- **Description**: Technical details of the vulnerability.
- **Impact**: Potential consequences and attack vectors.
- **Reproduction**: Step-by-step guide or proof of concept.
- **Environment**: Affected URL, browser, and OS (if applicable).

### Response SLA

- **Acknowledgment**: Within 48 hours.
- **Assessment**: Initial severity assessment within 5 business days.
- **Resolution**: Mitigation or remediation of critical vulnerabilities within 30 days.

## Disclosure Policy

Upon validating a vulnerability, we will:

1.  Collaborate with the reporter to verify the fix.
2.  Publish a security advisory on GitHub.
3.  Merge the patched code into the `main` branch.
4.  Credit the reporter in the advisory (unless anonymity is requested).

---

**Note**: This project does not currently operate a financial Bug Bounty program.
